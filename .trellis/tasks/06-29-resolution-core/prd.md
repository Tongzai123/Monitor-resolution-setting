# 分辨率切换底层闭环

## 目标

建立分辨率/刷新率功能的底层可靠闭环：读取当前模式、枚举可用模式、对单台显示器应用模式、确认倒计时和自动回滚。

## 需求

- 复用并扩展现有 `win32-displayconfig` 能力。
- 为每台活动显示器提供当前分辨率、刷新率和稳定识别字段。
- 枚举 Windows 报告可用的显示模式。
- 对指定显示器应用指定模式。
- 应用前保存原始模式和必要的原始显示配置。
- 主进程维护待确认切换状态。
- 用户未确认时自动还原原始模式。
- 切换失败和还原失败必须返回可展示错误。
- 普通切换失败只做面板/通知提示；自动回滚失败允许系统级对话框强提醒。

## 验收标准

- [ ] 至少一台真实显示器能读取当前 `width`、`height`、`refreshRate`。
- [ ] 能枚举该显示器可用模式，并包含当前模式。
- [ ] 能切换到一个可用模式。
- [ ] 15 秒未确认时自动还原原始模式。
- [ ] 面板关闭不影响自动回滚。
- [ ] 多显示器环境下不会对错误显示器应用模式。
- [ ] 自动回滚失败时弹出系统级对话框，提示用户手动恢复显示设置。
