# 分辨率显示与切换完整首版

## 目标

在 Twinkle Tray 现有托盘显示器控制体验中，增加完整的分辨率和刷新率显示、枚举、切换、确认回滚、收藏、设置、右键菜单和快捷键能力，让用户可以安全地在常用显示模式之间快速切换。

完整首版不是替代 Windows 显示设置中心；它只覆盖单台显示器的分辨率/刷新率快速切换，并通过确认倒计时降低黑屏和误选风险。

## 已确认事实

- 项目为 Electron + React 应用，主进程入口是 `src/electron.js`，托盘面板 UI 在 `src/components/BrightnessPanel.jsx`。
- 项目已依赖本地原生模块 `win32-displayconfig`，并在主进程中使用 `VerticalRefreshRateContext` 和 `addDisplayChangeListener`。
- `src/Monitors.js` 已通过 `w32disp.extractDisplayConfig()` 读取 Win32 显示器配置，并把部分 Win32 信息合并进现有 monitor 对象。
- 现有 `win32-displayconfig` 支持 `QueryDisplayConfig`、显示变更监听、显示启用/禁用、配置保存与恢复；没有现成的 `EnumDisplaySettingsEx` / `ChangeDisplaySettingsEx` 模式枚举与切换封装。
- 用户要求“初版内容，把五个阶段所有内容全部都要做”。

## 范围

### 必做

- 枚举每台显示器当前分辨率和刷新率。
- 枚举每台显示器支持的分辨率、刷新率组合。
- 在托盘主面板中显示当前分辨率，例如 `3840x2160 · 60Hz`。
- 分辨率入口采用类似 Windows 11 快捷设置的分段展开组件：左侧显示当前分辨率/刷新率，右侧小箭头用于展开模式列表。
- 支持对单台显示器切换分辨率和刷新率。
- 切换后显示确认倒计时，用户未确认则自动回滚。
- 支持收藏常用分辨率。
- 设置页提供分辨率控制总开关和模式筛选选项。
- 托盘右键菜单显示常用/收藏模式。
- 支持快捷键切换收藏模式。
- 预留场景预设数据结构，但不要求首版实现完整场景编辑和自动场景。
- 最终验收必须产出可直接双击运行的 Windows `.exe` 安装包或可执行产物，并记录输出路径。

### 不做

- DPI 缩放比例切换。
- HDR 开关、色深、颜色格式、动态范围切换。
- 显示器排列、主屏、副屏、复制、扩展模式管理。
- 按应用自动切换分辨率。
- 游戏启动检测和自动切换。
- 云同步或跨设备同步。

## 子任务

- `06-29-resolution-core`：底层显示模式枚举、应用、确认回滚闭环。
- `06-29-resolution-panel`：托盘主面板显示和下拉切换 UI。
- `06-29-resolution-settings-favorites`：设置项、收藏管理和持久化。
- `06-29-resolution-shortcuts-menu`：托盘右键菜单和快捷键切换。
- `06-29-resolution-integration-check`：完整首版集成验收、回归验证和收尾。

## 验收标准

- [ ] 能正确显示至少一台真实显示器的当前分辨率和刷新率。
- [ ] 能枚举每台已连接显示器的可用分辨率/刷新率组合，并标识当前模式。
- [ ] 主面板分辨率组件显示当前模式，右侧箭头可展开所有可选模式。
- [ ] 能对单台显示器切换到一个 Windows 报告可用的模式。
- [ ] 切换成功后出现确认倒计时；未确认时自动还原原始模式。
- [ ] 面板关闭或失焦后，确认倒计时和自动回滚仍然有效。
- [ ] 用户可以收藏常用模式，收藏按显示器持久化，重启应用后仍可用。
- [ ] 用户可以通过设置关闭主面板分辨率入口，并配置低刷新率/全部模式显示策略。
- [ ] 托盘右键菜单只显示常用/收藏模式，不展开完整模式列表。
- [ ] 快捷键可以切换收藏模式，并不破坏现有亮度快捷键。
- [ ] 现有亮度、HDR SDR 亮度、DDC/CI 功能不因该功能回归。
- [ ] 最终打包产物包含可直接双击运行的 `.exe`，并在验收记录中写明产物路径。

## 风险

- 错误模式可能导致黑屏、无信号、远程桌面断连。
- 显示器 ID 如果依赖连接顺序，插拔或多屏环境中可能错配。
- Windows 返回的 59.94Hz、60Hz、119.88Hz 等刷新率需要规范化显示但保留精确应用参数。
- 虚拟显示器、远程桌面、投屏设备可能枚举异常或拒绝应用模式。

## 产品决策

- 普通切换失败只在托盘面板或通知中提示。
- 自动回滚失败允许弹出系统级对话框提示，因为该情况可能导致用户无法正常恢复显示。
