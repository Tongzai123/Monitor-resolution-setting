# 分辨率显示与切换完整首版实施计划

## 阶段顺序

1. `06-29-resolution-core`
   - 扩展/验证 Win32 模式枚举和切换。
   - 合并当前分辨率/刷新率到 monitor 数据。
   - 实现主进程确认回滚状态机。

2. `06-29-resolution-panel`
   - 主面板显示当前分辨率/刷新率。
   - 下拉模式列表和切换入口。
   - 确认条和倒计时状态展示。

3. `06-29-resolution-settings-favorites`
   - 设置默认值、设置页、收藏持久化。
   - 模式筛选：低刷新率、全部模式、只显示收藏。

4. `06-29-resolution-shortcuts-menu`
   - 托盘右键菜单常用模式。
   - 快捷键切换收藏模式。

5. `06-29-resolution-integration-check`
   - 全链路验证、回归检查、文档和收尾。
   - 打包生成可直接双击运行的 Windows `.exe` 最终产物，并记录输出路径。

## 验证命令

- `npm run parcel-build`
- `npm run electron-build`
- `npm run dev`
- 需要原生模块变更时运行相关 rebuild 流程。

## 手工验证

- 单显示器：当前模式显示、模式枚举、切换、确认、自动回滚。
- 多显示器：确认目标显示器正确，收藏互不干扰。
- 面板关闭：倒计时继续并自动回滚。
- 低刷新率：默认隐藏，打开设置后可见。
- 右键菜单：只展示收藏/常用模式。
- 快捷键：不破坏现有亮度快捷键。
- 最终产物：生成 `.exe`，确认文件存在且路径写入验收记录。

## 回滚点

- 原生模块扩展失败时，可先只显示当前模式并暂停切换 UI。
- UI 接入异常时，可通过总开关隐藏入口。
- 快捷键或右键菜单不稳定时，不影响主面板切换能力。
